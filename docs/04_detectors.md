# 4. Детекторы ПДн (Detectors)

Сервис `PDNDetectors` (`app/services/detectors.py`) инкапсулирует логику поиска совпадений по переданным строкам.

## Поддерживаемые типы

Типы загружаются из `RegexRule` (таблица `regex_rules`):

| Тип | `pdn_type` | Описание |
|-----|-----------|----------|
| Phone | `phone` | Мобильные и городские номера |
| Email | `email` | Адреса электронной почты |
| Credit Card | `card` | Номера банковских карт (PAN) |
| FIO | `fio` | Русские ФИО |

## Интеллектуальный алгоритм поиска ФИО

Класс `PDNDetectors` содержит следующие вспомогательные методы:

| Метод | Описание |
|-------|----------|
| `detect(cls, text, field_path, rules)` | Основной метод, применяющий регулярные выражения и исключения |
| `_is_surname(word, surn_ends)` | Проверка окончания слова на фамильные суффиксы |
| `_is_patronymic(word, patron_ends)` | Проверка окончания слова на отчественные суффиксы |
| `_is_initials(word)` | Распознавание формата `И.О.` или `И. О.` (regex: `^[A-ZА-ЯЁ]\.\\s?[A-ZА-ЯЁ]\.$`) |
| `analyze_fio(text, surn_ends, patron_ends)` | Основной метод поиска ФИО — токенизация текста и проверка по паттернам |

### Поддерживаемые порядки ФИО

Метод `analyze_fio` проверяет по нескольким паттернам одновременно:

1. **Фамилия Имя Отчество** — 3 слова, w1=фамилия, w3=отчество
2. **Отчество Фамилия Имя** — 3 слова, w1=отчество, w2=фамилия
3. **Имя Фамилия Отчество** — 3 слова, w2=фамилия, w3=отчество
4. **Фамилия И.О.** — 2 слова, w1=фамилия, w2=инициалы
5. **И.О. Фамилия** — 2 слова, w1=инициалы, w2=фамилия

Для группы из 2-3 последовательных токенов (заглавные слова) каждый сверяется со словарями фамилий и отчеств по окончаниям.

## Фильтрация по исключениям (Exclusions)

Типы исключений из `RegexRule` (`rule_type`):

| `rule_type` | Описание |
|-------------|----------|
| `regex` | Системный детектор (основной паттерн) |
| `exclude_pattern` | Глобальное исключение JSON-пути |
| `prefix_exclude` | Исключение по префиксу значения |
| `suffix_exclude` | Исключение по суффиксу значения |
| `exclude_key` | Исключение конкретного ключа |
| `invalid_def_code` | Невалидный DEF-код (для телефонов) |
